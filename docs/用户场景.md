## 2. 通过邮箱注册账号

<a id="toc-2-1"></a>

### 2.1 提交注册信息后提示验证邮件已发送

#### 步骤

1. 打开网站上的注册页面，填写用户名、邮箱、密码及确认密码。
2. 提交后，界面应出现成功类提示（如验证邮件已发送）；系统会校验信息并向你的邮箱发送一封验证邮件（内含验证链接）。

#### 在邮箱中完成验证（人工，无 E2E）

1. 在邮箱中点击验证链接，按页面提示完成验证。
2. 验证通过后按第 3 章完成登录；忘记密码见 §3.1「找回密码」。

<a id="toc-3"></a>

## 3. 登录

<a id="toc-3-1"></a>

### 3.1 通过邮箱登录

#### 步骤

1. 打开网站上的登录页面。
2. 输入已验证邮箱与密码并登录。
3. 失败时按页面提示检查密码与邮箱验证状态。

#### 找回密码

**步骤**

1. 在登录相关流程中进入忘记密码页面，输入注册时使用的邮箱并提交。
2. 查收重置密码邮件并按链接打开页面（无对应账号时，界面可能不说明是否存在账号）。
3. 在页面填写邮箱、新密码与确认密码并提交。
4. 返回登录页用新密码登录；本机原登录状态可能被清除。

---

#### 通过邮箱免密码登录（无 E2E）

**步骤**

1. 打开网站上的邮箱登录 / Email sign-in 入口，输入邮箱并请求发送登录链接。
2. 打开邮件中的链接完成登录。
3. 提示「请查收邮件」时，留意收件箱与垃圾箱。

---

#### 通过 GitHub 登录（无 E2E）

**步骤**

1. 在登录页点击 Login with Github（或同等入口）。
2. 在 GitHub 完成授权后返回本站。

## 5. 创建空白项目并使用基础功能

与路线图「创建空白项目并使用基础功能」及 Playwright `test.describe` 标题一致；产品界面中项目类型常为 **数学建模 / Modeling**。**5.1～5.13** 见 `reaslab-test/test/05-blankModeling.test.ts`。

<a id="toc-5-1"></a>

### 5.1 创建空白 Modeling 项目

#### 步骤

1. 登录后在工作台左侧选择 New Project（新建项目）。
2. 填写项目名称，选择项目类型：
   - **数学建模**：获得适合写 Python 与说明文档的默认工程。
3. 创建后进入编辑环境。

---

<a id="toc-5-2"></a>

### 5.2 上传图片

经 ReasLingo 侧上传入口，将 **`reaslab-test/test/data/test_upload.png`** 写入项目（常见路径为 **`chat-uploads/test_upload.png`**），并在文件树或输入区芯片中可见。

---

<a id="toc-5-3"></a>

### 5.3 使用OCR进行AI会话

在默认 Agent 下：等待图片 **OCR Processing** 结束，将模型切为 **Auto**（若界面提供），用**全英文**就图中问题提问，待本轮助理回复结束；界面不应持续出现 **OCR Failed**，且应对图中算术给出正确数字答案（自动化用例校验为 **4**）。

---

<a id="toc-5-4"></a>

### 5.4 默认 Agent 进行 AI 会话

保持 **默认 Agent**（输入条按钮文案为 **Agent**；`AgentSelector` 下拉中**不**展示默认项，仅列 Math Modeling、ReasFlow Copilot 等可选 Agent）。关闭 **Auto**（若 §5.3 已开启）后，发送 **「who are you?」** 一条消息，待本轮流式回复结束（**不**再走看图 / OCR，该路径由 §5.3 覆盖）。

---

<a id="toc-5-5"></a>

### 5.5 邀请他人共同编辑项目

2. 生成分享链接发给对方；对方登录后打开链接验证即可进入。
3. 长期管理成员与权限：设置 → 协作者。

2. 选中快照查看文件与说明；若有「检出 / 恢复」，可按提示切换工作区。
3. 用差异对比查看与当前或其他版本的差别。

<a id="toc-5-9"></a>

### 5.9 AI会话设置（Chain of Thought）

在 **ReasLingo** 输入区底部（**发送**按钮左侧）、且当前为 **默认 Agent** 时，可出现 **Chain of Thought** 入口（灯泡图标，`title` 多为 **「Chain of Thought」**）。用于调节推理链展示强度（如 **None / Low / Medium / High**）。**仅在当前有效解析模型**（**`ChainOfThoughtSelector`** 中 `resolveStorageModelForCapabilities`）**声明 `supportsReasoning` 时渲染**：**Auto 开启**时有效模型为**系统默认模型**——若默认模型支持推理，**Auto 下即可见灯泡**；**Auto 关闭**后则看手动选中的模型是否支持推理。自动化用例**先检测灯泡是否已可见**（兼容 Auto + 默认可推理），不可见时再关 **Auto** 并遍历启用模型。

**自动化**：**`test/05-blankModeling.test.ts`** **5.9**；若环境**未启用**任何 **`supportsReasoning`** 模型（模型菜单除 **Auto** 外无项或遍历后仍无 CoT），**`test.skip`**（**`MODELING_CH5_CHAIN_OF_THOUGHT_SKIP_MSG`**），不记为失败。

与 **§7.6**「齿轮」类全局 AI 设置区分：本节仅描述 **输入条上的会话侧快捷项**。

#### 步骤（人工验收建议）
