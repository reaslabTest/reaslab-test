## 2. 通过邮箱注册账号

<a id="toc-2-1"></a>

### 2.1 提交注册信息后提示验证邮件已发送

#### 步骤

1. 打开网站上的注册页面，填写用户名、邮箱、密码及确认密码。
2. 提交后，界面应出现成功类提示（如验证邮件已发送）；系统会校验信息并向你的邮箱发送一封验证邮件（内含验证链接）。

#### 在邮箱中完成验证（人工，无 E2E）

1. 在邮箱中点击验证链接，按页面提示完成验证。
2. 验证通过后按第 3 章完成登录；忘记密码见 §3.1「找回密码」。

---

<a id="toc-3"></a>

## 3. 登录

<a id="toc-3-1"></a>

### 3.1 通过邮箱登录

#### 步骤

1. 打开网站上的登录页面。
2. 输入已验证邮箱与密码并登录。
3. 失败时按页面提示检查密码与邮箱验证状态。

#### 找回密码

**步骤**

1. 在登录相关流程中进入忘记密码页面，输入注册时使用的邮箱并提交。
2. 查收重置密码邮件并按链接打开页面（无对应账号时，界面可能不说明是否存在账号）。
3. 在页面填写邮箱、新密码与确认密码并提交。
4. 返回登录页用新密码登录；本机原登录状态可能被清除。

---

#### 通过邮箱免密码登录（无 E2E）

**步骤**

1. 打开网站上的邮箱登录 / Email sign-in 入口，输入邮箱并请求发送登录链接。
2. 打开邮件中的链接完成登录。
3. 提示「请查收邮件」时，留意收件箱与垃圾箱。

---

#### 通过 GitHub 登录（无 E2E）

**步骤**

1. 在登录页点击 Login with Github（或同等入口）。
2. 在 GitHub 完成授权后返回本站。

## 5. 创建空白项目并使用基础功能

与路线图「创建空白项目并使用基础功能」及 Playwright `test.describe` 标题一致；产品界面中项目类型常为 **数学建模 / Modeling**。**5.1～5.8** 见 `reaslab-test/test/05-blankModeling.test.ts`。

<a id="toc-5-1"></a>

### 5.1 创建空白 Modeling 项目

#### 步骤

1. 登录后在工作台左侧选择 New Project（新建项目）。
2. 填写项目名称，选择项目类型：
   - **数学建模**：获得适合写 Python 与说明文档的默认工程。
3. 创建后进入编辑环境。

---

<a id="toc-5-2"></a>

### 5.2 上传图片

经 ReasLingo 侧上传入口，将 **`reaslab-test/test/data/test_upload.png`** 写入项目（常见路径为 **`chat-uploads/test_upload.png`**），并在文件树或输入区芯片中可见。

---

<a id="toc-5-3"></a>

### 5.3 使用OCR进行AI会话

在默认 Agent 下：等待图片 **OCR Processing** 结束，将模型切为 **Auto**（若界面提供），用**全英文**就图中问题提问，待本轮助理回复结束；界面不应持续出现 **OCR Failed**，且应对图中算术给出正确数字答案（自动化用例校验为 **4**）。

---

<a id="toc-5-4"></a>

### 5.4 切换 ReasLab Agent 进行 AI 会话

将 Agent 切换为 **ReasLab Agent**；发送 **「who are you?」** 一条消息，待本轮流式回复结束（**不**再走看图 / OCR，该路径由 §5.3 覆盖）。侧栏 Agent 菜单须含 **ReasLab Agent**，否则对应 E2E 失败。

### 5.5 邀请他人共同编辑项目

2. 生成分享链接发给对方；对方登录后打开链接验证即可进入。
3. 长期管理成员与权限：设置 → 协作者。

### 5.6 查看项目的修改历史

2. 选中快照查看文件与说明；若有「检出 / 恢复」，可按提示切换工作区。
3. 用差异对比查看与当前或其他版本的差别。

## 7. 模板创建优化建模项目

**7.3** 依赖侧栏存在 **Optimization Agent**，否则 E2E 会跳过该步。**7.5** 在 **默认 Agent** 下依赖 ReasLingo 能经 **MCP** 调用 **`python_mcp`**（与 **§7.4** 工具栏 **Run Python** 路径不同）。**7.6～7.7** 为 **ReasLingo** 历史会话与 **Settings（齿轮）** 内模型/MCP 等配置验收。**E2E**：**7.1～7.7** 见 **`reaslab-test/test/07-optimizationTemplateModeling.test.ts`**（**7.7** 与 **`helpers.ts`** **`reasLingoIdeSettingsAiFlow`** 对齐）。

<a id="toc-7-1"></a>

### 7.1 从优化建模模板创建项目并进入建模 IDE

#### 步骤

1. 在工作台选择 Optimization Modeling Templates（优化建模模板）。
2. 浏览模板，进入详情后使用「从模板创建」：系统会新建数学建模类项目，并写入模板中的代码与说明等内容。
3. 创建完成后进入项目编辑环境。

<a id="toc-7-2"></a>

### 7.2 打开 README.md 并显示 Markdown 预览

打开的为项目根目录 `README.md`。

#### 步骤

1. 进入项目后打开 README.md，点击预览（眼睛图形）显示预览内容。

<a id="toc-7-3"></a>

### 7.3 ReasLingo：Optimization Agent 与 who are you?

#### 步骤

1. 侧栏打开 ReasLingo，Agent 选 Optimization Agent、发送「who are you?」一条消息，待本轮流式回复结束后再继续。

<a id="toc-7-4"></a>

### 7.4 模板内 Python：打开主脚本、Run Python 与 Console 验收

在已从 **Optimization Modeling Templates** 进入的建模 IDE 内，对模板自带的求解 **`.py`** 使用底部 **Console** 与工具栏 **Run Python** 验收运行结果（与 **`bottom-panel.tsx`** 一致：stdout / stderr 分区展示，**最终以状态行 `Process finished with exit code N` 为准**）。**端到端**见 **`reaslab-test/test/07-optimizationTemplateModeling.test.ts`** 中 **7.4**（复用 **§7.1** 进入的模板项目）。

#### 步骤

1. **打开脚本**：在 **Explore** 文件树中打开模板内的主 **`.py`**（如 **1-M-1** 类模板根目录下的求解脚本；若需读 **`data/`**、写 **`images/`**，以模板仓库自带目录为准，**不必**再手工上传同名文件）。
2. **运行与面板**：底部切换到 **Console**；编辑器工具栏点击 **运行**（**`lucide-play`** / **Run Python**，与 **`Hotkey.RUN_PYTHON_FILE`** 一致）。
3. **Console 验收**：底部状态行 **`Process finished with exit code 0`**（**`N === 0`**）→ **通过**；**`N !== 0`** → **不通过**（stderr 中常见 **`GurobiError`** / **`Unauthorized access`** 等；**E2E** 对许可证类失败可 **`test.skip`** 以便无 Gurobi 许可的 CI 不标红）。长日志时该行在 **Console** 最下方，**E2E** 会将 **`ScrollArea`** 视口滚到底再读全文。

<a id="toc-7-5"></a>

### 7.5 使用 MCP 执行 Python

在 **§7.1** 已进入的同一优化建模模板项目中，通过 **ReasLingo**、**默认 Agent**（**不**切换为 Optimization 等其它 Agent）在对话中调用 **`python_mcp`**（**Model Context Protocol**），由后端 **MCP 服务**在**项目隔离环境**内执行 Python 相关工具（以产品当前暴露的工具为准；设置里 **ReasLingo → Tools & MCP** 可见 **`python_mcp`** 说明，与 **`ReasLingoSettings.tsx`** 一致）。

与 **§7.4** 的差别：**7.4** 是用户在编辑器内点 **Run Python**，进程由 IDE **Console** 面板承接；**7.5** 是模型在聊天流中 **发起工具调用**，经 **MCP** 与运行环境交互。同一模板主 **`.py`** 在 **E2E** 中**先后执行两遍**：**先** **§7.4** 运行键，**再** **§7.5** **`python_mcp`** 全量执行同一路径，用于「IDE Console + AI MCP」双路径验收。

#### 步骤

1. 侧栏打开 **ReasLingo**，保持 **默认 Agent**（**E2E** 不切换 Agent；**§7.3** 仍单独验收 **Optimization Agent**）。
2.（可选）进入 **ReasLingo 设置**，打开 **Tools & MCP**，确认 **`python_mcp`** 为已安装且可用状态（绿点）。
3. 在完成 **§7.4**（同一项目、同一主 `.py` 已由 **Run Python** 跑过）后，在输入区发送一条**明确要求经 `python_mcp` 再次全量执行该 `.py`**（与 **§7.4** 打开的脚本路径一致）的指令。等待本轮回复结束。
4. **验收**：回复/工具输出中应出现与 **§7.4 Console** 一致的成功语义（如 **`Process finished with exit code 0`**）；若界面有工具调用轨迹（展开/「used tool」类 UI，以线上为准），应能看到与 **`python_mcp`** 相关的调用记录。若环境禁用 MCP 或模型未发起工具调用，记为**不通过**并记录原因。

<a id="toc-7-6"></a>

### 7.6 切换 AI 历史会话

在 **§7.1** 已进入的同一优化建模模板项目中，于侧栏 **ReasLingo** 内切换**历史聊天会话**，并用一句话验证当前会话上下文是否为所选会话（与输入框清空/新建会话区分）。

**前置（与 §7.1～7.5 衔接）**：按主线走完 **§7.1～7.5** 后，侧栏 **Chat History** 中通常会出现 **至少两条会话**（例如 **§7.3** 中「who are you?」一条、**§7.5** 中 **`python_mcp`** 一条；具体标题以产品生成为准）。**§7.6** 依赖上述多条会话；若仅有单条或列表为空，切换与回放验收无意义。

### 12.1 上传本地 LaTeX 并编译预览
